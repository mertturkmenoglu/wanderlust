# ADR-0001: Use Architectural Decision Records

## Context
We want to capture the tech choices, constraints, tradeoffs we made. Architectural Decision Records (ADR) help with this issue.
There are multiple ADR formats, most notable ones are:

- Nygard
- MADR
- Y-Statement
- Outcome-First
- ISO 42010

A comparison of the formats can be found here: https://www.adr.zone/compare-formats

## Decision
We choose the Nygard format for our ADRs. It is simple and easy to read.

**Addendum**

We modified the Nygard format and created our custom template. The template can be found here: [ADR Template](./template.md)

## Consequences
- Positive: 
	- Easy to read and understand.
	- Provides a clear structure for documenting decisions.
- Negative: 
	- Introducing any ADR format will require some learning and adaptation for team members.
	- We must ensure that decisions are documented consistently and kept up to date.
